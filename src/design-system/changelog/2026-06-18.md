## 按钮 · 投影体系（已 DevTools 验证通过）

- 厘清投影是按 variant × state 的条件矩阵：一级实心 / 二级描边（含 dangerSecondary）仅 hover 有投影 = D3↓；三级 / 虚线 / 链接 / 链接下拉任何状态无投影；悬浮按钮所有状态恒有 D4↓；其余状态（default/active/disabled/loading）无投影，禁用/加载强制无投影。
- 投影色统一走 `mask-01-transparent`（#0B0C0D）+ `alpha()` 派生，禁止裸 `rgba(11,12,13)`；投影定义集中在 `build-tokens.mjs`（D1/D3/D4/None）。
- **关键坑（已解决）**：上轮写了 `primaryShadow:"none"` 但真实 hover 仍冒绿影。DevTools 抓到 box-shadow 是 `rgba(8,168,95,.15)`，定位为 antd Button 默认 `primaryShadow ← controlOutline ← colorPrimaryBg`（浅绿）。修复：在 `components.Button` 把三类 shadow + 全部 PresetColor ShadowColor 清零，不动全局 `colorPrimaryBg/controlOutline`；D3/D4 由 `SensButton` 条件 inline 注入。
- **验收标准（已立为规矩）**：不信「已执行」自检，改完用 DevTools 抓 hover 的 box-shadow，颜色必须是 `rgba(11,12,13,…)`，不得是 `rgba(8,168,95,…)`。已验证通过。
