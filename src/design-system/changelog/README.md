| 日期 | 文件 |
|------|------|
| 6月22日 | [2026-06-22.md](./2026-06-22.md) |

| 6月17日 | [2026-06-17.md](./2026-06-17.md) |

## 写法模板

每天一个文件：`changelog/YYYY-MM-DD.md`。

```markdown
# 设计还原产线 · 更新日志 · M月D日

> 记录对「设计还原产线」和 `src/design-system` 知识库做的事。

## [组件名 / 横切主题]（可选副标题：第 N 轮 / 修复）

- [决策] 做了什么、为什么（1–2 句 why，可合并 bullet）
- [实现] 动了哪些文件 / 规则（点到名为止）
- [token] 引用 `theme.ts` / `build-tokens.mjs` 中的 token 名或语义 handle；**不写死 hex**
- [坑] 踩了什么、怎么避（可选）
- [验收] ✅已验（注明验法，如 Playwright computed / 矩阵四态）| 🔶待验 | ⏳待做

## 待办 · 未完成欠账（仅写在最新一天）

- …
```

**结构约定**

| 层级 | 规则 |
|------|------|
| 文件标题 | `# 设计还原产线 · 更新日志 · M月D日` |
| 范围块 | 首段 `>` 说明当日记录范围 |
| 正文 | 按**组件或横切主题**用 `##` 二级标题分组 |
| 条目 | 每条须覆盖：**改了什么 + 决策/why**；有坑写坑；涉及色/距写 **token 名** |
| 验收 | 必须用明确标记：`✅已验「验法」` / `🔶待验` / `⏳待做` |
| 横切 | 单独 `##` 节；须附 **回归验了什么**（组件/路由/换肤等） |
| 待办 | **只写在最新一天**；前一天视为已归档 |

## 触发阈值（何时必须写）

当轮收尾前，满足**任一**即须补当日 changelog（可多条 bullet，不必等「大轮」）：

1. **已落地且已验**：代码/样式/token 映射已合入，并有明确验收（含 Playwright、矩阵、对照截图等）
2. **做出或推翻决策**：方案选型、规则口径变更、弃用某实现路径——即使用户未点名「写日志」

**不算触发**：纯讨论未落地、单 keystroke 措辞微调、仅读代码未改、用户明确说「本轮不写日志」。

**含小修正**：token 名修正、尺寸 1px、压边/接缝类修复——只要已落地（+已验或决策已定），**算触发**。

## 责任方

| 事项 | 谁做 |
|------|------|
| 落盘 `changelog/YYYY-MM-DD.md`、按模板补条目 | **Cursor**（当轮收尾） |
| 决策 / why / 是否记为「已验」 | **用户**收口时提供；Cursor 整理进条目 |
| **commit** | **Cursor**：changelog **必须与当轮代码/样式改动进同一个 commit**，不得单独留空或等用户另开 commit |

## 完成定义（DoD）

写一条 changelog **不算完成**，须同时满足：

1. 已**新建或更新** `src/design-system/changelog/YYYY-MM-DD.md`，且条目符合上文模板
2. 打开 `/changelog` 预览，**Segmented 自动出现该日期**（`ChangelogPage` 通过 `import.meta.glob` 扫 `changelog/20*.md`，**无需**改 `ChangelogPage.tsx` 注册表）
3. 选中该日期后正文渲染正常

**维护（索引表）**：当天收尾时新建/更新当日 md；上表「日期 | 文件」**人工补一行**便于仓库内跳转（预览不依赖此表）。
