# 设计还原产线 · 更新日志 · 6月24日

> 记录对「设计还原产线」和 `src/design-system` 知识库做的事。

## 按钮 · 悬浮按钮（FAB）完整体系：单项 + 横向组合 + 竖向组合

- [决策] FAB 分单项与组合，组合不嵌单项：直接拼 `fab=true` 单项会带进 per-button D4、全周 999、错 padding，与组合规格冲突。组合走自定义容器 + 裸 antd Button 分段，共用单项配色/状态语义（同源）、几何/投影/padding 独立实现。公共部分抽 `fabShared.ts` 防配色漂移。不用 `Space.Compact`：其 `margin-inline-end: -1px` 是为有描边控件消双线（搜索缝那套），填色块齐平用不上、反致位移。
- [实现·单项] 单项 FAB 横向 padding 由继承的 12 改为 FAB 专属 `spacing/horizontal/5x`=20，scoped 仅 `fab=true` 生效（走 `paddingInline`）；全局 `components.Button.paddingInline` 保持 12 不动（Input 同源 + 搜索缝依赖）。纯图标单项为圆形。
- [坑] 覆盖单项 padding 必须用 `paddingInline`，只设 `paddingLeft`/`Right` 会被 antd 主题的 `paddingInline: 12` 盖回——antd 默认行为暗坑。
- [实现·横向组合] 新建 `SensFabGroup`：2~3 独立可点按钮齐平拼胶囊，`tone` 在组合级（不混级）；外端 999 内缝 0；每按钮外 20（`spacing/horizontal/5x`）内 10（`SP_H_2_5`）；整组一层 D4 在容器、子段无投影；配色一级绿底白字（hover 底变深）、二级白底黑字无描边（`variant=text` + 窄域锁白底，hover 字转 `component-hover`）。直读 `tokens.resolved.json`（2.5x 的 key 含 U+2024 点号，已校验取值 10）。
- [实现·高度修正] FAB 交叉轴高度由初版误用的 `controlHeight`=32 改为 FAB 专属 `size/component-height/l`=36（纯图标单项圆 36×36；横向组合每段 36），走 `--sens-fab-height`、scoped 到 FAB 形态、不动 `controlHeight`，常规 Button 仍 32。
- [实现·竖向组合] `SensFabGroup` 加 `direction` 轴扩竖向：几何层 `direction` 参数化（横向 default 行为冻结）、配色层竖向独立、拆两个 Segment 共用几何 helper。竖向只二级 + 纯图标 + 白底无描边、交叉轴宽 36、图标 `size/icon/m`=16、上下端 999 内缝 0、外 20 内 10（顶底）左右 0、整组一层 D4、最多 3 个、只 default/hover/active（无禁用/加载，类型层用 discriminated union 锁死竖向不出现 `tone`/`disabled`/`loading`）。
- [决策·配色独立] 竖向图标色阶【不复用横向二级】：默认 `icon-color-transparent`（图标色，非横向的 `text-color`）、hover `component-primary`（默认绿，**非**横向 hover 的 `component-hover`）、active `component-active`、全态白底。hover/active 引 `component-primary`/`active` 链换肤、不写死。
- [token] `spacing/horizontal/5x` → 单项/组合外 padding 20；`spacing/horizontal/2․5x`（`SP_H_2_5`）→ 组合内 padding 10；`size/component-height/l` → `--sens-fab-height` 36；`size/icon/m` → 竖向段内图标 16；`radius/circular` → 999；`mask-01-transparent` → D4。
- [验收] ✅已验「DevTools computed」：横向 segment 36、padding 20/10、gap 0、整组 D4；单项 FAB 文字 36 / 纯图标 36×36、padding 20；常规 Button 仍 32；竖向容器宽 36、图标 16×16、首 20/10 中 10/10 末 10/20 左右 0、首末 999 内缝 0、整组一层 D4 子段无投影；竖向默认 `icon-color-transparent` / hover `component-primary`（非 `component-hover`）/ active `component-active`、`variant=text` 抢色已被竖向色链压过没被 antd 默认盖回；`--sens-fab-height` / `--sens-fab-icon-size` 均 token 非硬编码；改共用几何层后横向单项 + 组合零回退。

## 按钮 · 横向组合 FAB padding 压盖修复（纯图标深色带）

- [坑] 组合分段只设 `paddingLeft`/`paddingRight` 会被 antd 主题 `components.Button.paddingInline`=12 盖回（与单项 FAB 同坑）；纯图标段另加 `ant-btn-icon-only` 定宽 32px，错误 padding 挤在定宽盒内 → 中段可见深色带 + 间距不匀。
- [实现] `fabShared.ts` 增 `getFabGroupSegmentPaddingStyle`（横向走不对称 `paddingInline`）；`getFabGroupSegmentCrossAxisStyle` 对 `isFabIconOnly` 段加 `width:auto` 解开定宽；`SensFabGroup` 真实段与 `getFabGroupSegmentPreviewStyle` 预览段共用 helper。
- [token] 外 `spacing/horizontal/5x` 20、内 `spacing/horizontal/2․5x` 10；未动全局 `paddingInline` / 竖向 / 单项 / 一级 transparent border。
- [验收] ✅已验「Playwright computed + 截图 `/components/button` #sens-fab-group-preview」：全内容×全段数（纯图标/文字/图文 ×2/×3，一级+二级）段 padding 首 20/10、中 10/10、末 10/20；内缝 gap 0、整组 999 + D4；纯图标×3 三段齐平胶囊、深色带消失；文字/图文段宽 64/54/64、80/70/80 与规格一致（修前同为 12/12 被盖回，非锚旧宽 127.58）。
