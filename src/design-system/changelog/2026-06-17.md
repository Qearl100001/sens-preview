# 设计还原产线 · 更新日志 · 6月17日

> 记录对「设计还原产线」和 `src/design-system` 知识库做的事。

## 组件 skill

- 新增**标签页**规则：厘清三种标签页（基础 / 页签 / 胶囊）分别对应三个不同的底层组件，胶囊标签页不是按钮；为胶囊的选中态钉了专用颜色 token。
- 优化**标签页（按钮范式对齐）**：按「单一真实组件 + 单一预览组件」收敛，真实层仅保留尺寸/间距/滚动/禁用等交互必需；上半区（单项状态）与下半区（完整状态）统一改为同一个快照函数渲染，避免两块不同源导致的状态漂移；同时清理补丁式覆盖，`tabs.css` 从 **272 行收敛到 196 行**。
- 新增**徽标**规则：三种类型（常规型数字 / 状态点 / 弱化型）映射到位；发现徽标颜色大多能直接复用已有的状态色 token，几乎零新增。
- 优化**搜索**：清理了组件里大量对 antd 的平行 CSS 覆盖，改为主题驱动；图标透明度等改走 token。
- 修整**按钮**：修正了两类反复出现的状态问题——二级/三级 hover 被错误地加了背景填充、禁用与加载样式混乱；把规则按团队真实约定写死（二级 hover 无底、禁用与加载视觉一致）。

## 主题与颜色

- 明确**功能色 vs 状态色**的分层：功能色（神策绿系）会换肤，状态色（链接蓝、警告红、提醒黄、成功绿、涨跌色）不换肤；业务代码只引用 token 名、不写 hex、不引用基础色板。
- 链接按钮归位到状态色（colorLink），不再误用功能色。
- 按钮的状态色相关 token 经历「加错→删除→按团队规则改对」的修正，最终回到主题驱动、无手写覆盖。
