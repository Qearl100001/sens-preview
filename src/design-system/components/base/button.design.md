# 按钮 Button · 设计规范（给设计师）

> 这份讲"用什么、为什么、规格多少"，面向设计与评审。
> "用 antd 怎么实现"见研发文档 `button.md`，本文不涉及代码/组件名。

## 什么时候用按钮
按钮用于**触发一个明确的操作**（提交、创建、删除、打开…）。一个区域里按钮不宜过多，主操作只留一个最显眼的。

## 类型：按重要程度选（这是选型的核心）
| 类型 | 用在哪 | 视觉 |
|---|---|---|
| 一级（实心绿）| 页面/区域里**最重要的那一个**主操作 | 神策绿实心 + 白字 |
| 二级（绿描边）| 次要但常用的操作 | 白底 + 绿描边绿字 |
| 三级（绿文字）| 更弱的操作、密集场景 | 仅绿字、无边框 |
| 链接 | 行内、表格操作列等"像链接"的操作 | 蓝色文字（状态色，不随换肤变）|
| 警告 | 删除等**破坏性**操作 | 红色，建议配二次确认 |
| 虚线 | 在空区域"添加"内容 | 虚线框 + 加号 |

> 选型口诀：一个区域**只有一个一级**；越往下越弱；破坏性操作用警告色 + 二次确认。

## 尺寸
- 只有**大（32px，常用）/ 小（24px，空间紧张时）两种**。不要引入第三种尺寸。

## 颜色语义（只记角色，不记色值——色值是 token 的事）
- 主操作 = **功能色（神策绿）**，会随换肤变。
- 警告 = 旭日红、链接 = 冰绽蓝，属**状态色，不随换肤变**（详见设计系统颜色分层）。
- 二级 / 三级 hover、点击**只变文字/边框深浅，不加背景填充**（团队规则）。
- 禁用 与 加载呈现一致（同一套禁用灰），这是团队约定。

## 状态
默认 / 悬停 / 点击 / 禁用 / 加载，共 5 个语义状态（悬停态在禁用/加载下不再变化）。

## 图标
- **非必要不用图标**；用图标必须有明确价值。图标颜色与间距见图标规范。

## 文案
- 以**动词**为主（"保存""删除""创建"），简洁明确。

## 组合规则
- 多个按钮并排时按优先级**从右到左**排，主按钮最显眼。
- **操作 ≥ 4 个时收进「更多」**，避免一排太满。
