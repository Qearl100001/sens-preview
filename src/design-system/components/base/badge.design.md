# 徽标 Badge · 设计规范（给设计师）

> 这份讲"用什么、为什么、规格多少"，面向设计与评审。
> "用 antd 怎么实现"见研发文档 `badge.md`，本文不涉及代码/组件名。

## 徽标是什么
透传当前层级上**未显示的、关键的、临时**信息的一种**配合型**组件。它是个"引导"，承载信息不宜多。

## 三条铁律
- **必须依附其他组件**使用，不能独立做常驻标签。
- **临时性**：动作完成（查看/处理任务）后应及时消失，不允许长期稳定存在。
- **非必要不使用**（它是"破坏性"引导）；关键场景（必须引导用户去做某事）才鼓励用。

## 出现 / 消失
- 出现：要强调的关键状态发生时。
- 消失：被引导的任务被完成时。

## 三种类型（按"视觉破坏性"由强到弱选）
| 类型 | 用在哪 | 视觉 |
|---|---|---|
| 常规型数字徽标 | 强调关键数量变化，最需要被注意 | 红底白字数字，数字过大封顶（如 99+/999+，封顶值业务定）|
| 状态徽标（点）| 强调状态变更，不关心具体数量 | 一个小圆点，按语义取色（旭日红/原野黄/极光绿/冰绽蓝/子夜黑）|
| 弱化型数字徽标 | 数量变化但无须强破坏 | 灰底数字，依附在标签/入口上 |

### 点徽标 · 五色

| 稿面名 | 语义层级 | 说明 |
|---|---|---|
| 旭日红 | 状态色 / 警告 | 红 |
| 原野黄 | 状态色 / 提醒 | 琥珀黄 |
| 极光绿 | 状态色 / 成功 | 绿 |
| 冰绽蓝 | 状态色 / 链接 | 蓝 |
| **子夜黑** | **中性色 / 图标** | **徽标组件独有**；灰点，不是状态色 |

### 弱化灰底

| 场景 | 灰底 |
|---|---|
| 标签页右侧 | 透明灰（与行底融合） |
| 按钮右上角 | 实心 `background-01`（避免透出按钮描边） |

## 注意
- 弱化型徽标**不允许作为常驻标签使用**——徽标只对"关键的临时状态"做强调。
- 点徽标前四色为**状态色**（语义固定，不随换肤变）；**子夜黑为例外**，走中性图标色。
