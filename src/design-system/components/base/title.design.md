## 录入结论

标题作为独立基础组件录入，不直接并入表单。原因是 Figma 中 Title 已有独立组件页和 component set，并且包含通用标题、营销云 / 分析云 / SDH 专用标题、大小尺寸变体。

已有 `TitleBar` 继续承担页面 / 抽屉 / 面包屑标题栏职责；本组件命名为 `SensSectionTitle`，避免和页面壳标题混淆。

与表单相关的标题口径：

1. 灰条 / 绿条只用于**当前分组树最顶层**（含白卡内重置后的顶层）；其下层级无条。
2. 顶层辅助文案与标题主文案同一行、基线对齐；实现为 `description` 进入标题行并 `align-items: baseline`。
3. 灰底卡内标题仅 14px；16px 暂无场景。第二层 16px 无灰条直接用 `font-size/l` token。

## Figma 提取摘要

- 通用大尺寸：灰背景，圆角，标题 16 / 24 / semibold，可带选填、帮助、说明和右侧操作；对应表单分组树顶层。
- 通用小尺寸：灰背景，圆角，标题 14 / 22 / medium；业务组件内部；不得充当表单第二 / 三层；灰卡内若需条样式需与「仅 14px」口径一起看。
- 专用大尺寸：左侧绿色短条，标题 18 / 28 / semibold，无灰背景。
- 专用小尺寸：左侧绿色短条，标题 16 / 24 / semibold，只允许在业务组件内使用。
- 专用标题说明明确限制在营销云、分析云、SDH，其他线使用前需要确认。
- `64:55048` 样张中，顶层标题与「辅助文案信息」横排，辅助文案跟标题基线对齐。
- `69:26542`：白卡内可再出灰条并重置层级；灰卡内不出现白描边卡，标题仅 14px。

## 设计边界

- 通用标题可以承载右侧操作，右侧操作仍使用按钮基础组件。
- 专用标题不承载右侧操作，不替代通用标题。
- 帮助图标暂不落错误资产；等 SensD help icon 入库后再接入。
- 表单样板间中出现的标题组合，后续按表单样板间补，不反向扩大基础标题能力。
- 辅助文案是标题行内元素，不是标题下方的第二行说明块。
