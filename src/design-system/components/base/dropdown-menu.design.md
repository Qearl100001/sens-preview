## 一、定义　✅

集合了选项的容器，允许用户在有限的空间内从选项列表中「单选」或「多选」。

---

## 二、基础构成

用于「点击」或「悬停」触发的下拉操作，点击空白区域可收起；内容较多时可根据场景添加搜索、分组或采用其他特殊形式。

- 和「选择器」搭配：**点击**触发。✅
- 和「更多按钮」搭配：**悬停**触发。✅
- 选择区域为**整行可点击**。✅

构成四部分：搜索条 Search bar、统计区 Count、内容模块 Content（必要项）、操作条 Operation bar。

### 2.1 搜索条　✅

- 出现场景：当能预料到下拉中显示内容条数未来较多时，带有搜索条。
- 规则：使用「简易版搜索」规则（见《搜索》）。

### 2.2 统计区　✅

- 统计内容区呈现的前端信息条数，**建议枚举项 ≥ 9 条时出现**。
- 当有搜索条出现时，统计区也一定会出现。
- 使用搜索后，此处显示结果条数：「已找到 *** 条」。

### 2.3 内容模块（必要项）

- 用来呈现枚举项，通常出现单选 ✅、多选 ⏳、带 icon ⏳ 等情况。
- 排序规则：可按业务自定义（如创建时间倒序），或默认按字母 / 数字顺序排序；如出现明显分类，则使用带层级分组的下拉。🔶（自定义排序由调用方传入；带层级分组⏳）

### 2.4 操作条

- 整个下拉菜单中进行操作的 toolbar，右下角操作建议用「完成」或「保存」+「放弃」。
- **单选**场景尽量不出现操作条，让用户点选直接完成。✅
- **多选**场景会显示操作条。⏳
- 操作条类型：常规 / 左侧带复选框 / 左侧带操作栏 / 自定义添加新值。⏳（均依赖多选 / checkbox，未建）

## 三、设计原则

### 3.1 交互规则

- **下拉选择后的定位**：选择后再次点开，下拉列表枚举项按默认排序规则排列。✅
- **下拉出现的位置**：根据空间是否足够自适应弹出方向 ✅
  - 空间足够：下拉框在下方、与触发框左对齐（**推荐**）。
  - 右侧空间不足：下拉框在下方、与触发框右对齐。
  - 下方空间不足：下拉框在触发框上方、左对齐。
  - 下方 + 右侧均不足：下拉框在触发框上方、右对齐。
- **搜索的规则**：推荐用于前端请求的数据范围内；数据量过大无法前端展示时，推荐使用「穿梭框」。⏳（穿梭框未建）
  - 单选搜索流程：激活搜索（未输入前仍展示选项列表）→ 输入关键词筛选 → 选择后直接完成、收回菜单。✅
  - 多选搜索流程：激活 → 输入筛选 → 可点「返回」回到选项列表（带回所选）/ 可点「完成」收回；完成按钮数量 = 选中总数。⏳
- **选项状态**：当选项文案截断与禁用冲突时，悬停时优先展示**禁用说明文案**（tooltip）。🔶（禁用 hover 行为已还原；"禁用原因 tooltip" 待补）

## 四、类型

### 4.1 常规　✅

- 最常规的下拉类型，触发对象可以是**选择框、按钮、筛选**。
- 当内容区条数超出 9 条后出现滚动条，且显示成「9 条半」的样子，示意后面还有数据。🔶（滚动已还原；"9 条半"精确量待核）
- 覆盖场景：枚举项 < 9 时 / ≥ 9 时（带统计区）/ 显示搜索栏、操作栏时。✅
- **还原说明**：已还原两条触发路径——选择器点击触发（`SensSelectDropdown`）、更多按钮悬停触发（动作菜单 `SensDropdownMenu`，菜单项含中性黑 / 链接蓝 / danger 三档，行底走中性灰）。菜单项 variant 与触发器细节见研发文档。

### 4.2 级联　⏳

- 有多个枚举项，且其中枚举项带子级。适用于多层级（**≤ 3 级**）的情况，hover / 点击后展示子级。
- 含：单选的级联 / 多选的级联。

### 4.3 带层级　⏳

- 有多个枚举项，且有上下级关系，可通过**区块**或**线**进行分割。

### 4.4 双行　⏳

- 用于特殊场景，第一行为主要内容，第二行放置辅助信息；适用于需要在外面直接看到辅助信息的场景。

### 4.5 树形　⏳

- 应用于多层级的、带有结构的选择场景。

## 五、场景说明

### 5.1 多选场景复选框带禁用项　⏳

- 出现禁用选项时，其全选操作后可提示 tips，可业务线自定义。
- 状态覆盖：基础状态 / 禁用后 hover 显示禁用原因 / 禁用已选 / 禁用未选 / 禁用未选-已选都有。
- 全选遇禁用项时提示「部分选项状态不可更改」；含半选（indeterminate）态。
- 依赖 checkbox + 多选，均未建。

### 5.2 收回规则与计数规则

- **单选**：选中 / 更改选项即收回菜单、完成操作；点击空白处收回菜单、**不保存**结果。✅
- **复选**：操作完成后需点「完成」按钮收回菜单、结束操作；点击空白处收回菜单、**不保存**结果。⏳
- 例：进入时选 5 项，进来又减 2 项，点空白位置则不保存，再次进入仍是 5 项（多选场景，选择框内选项不变）。⏳

### 5.3 支持后端请求数据　⏳

- 数据量超大时，下拉菜单支持从后端请求数据；单选场景与前端一致。
- 多选 · 数据量大需后端加载：搜索时请求所有数据一起拉取；选中的后端数据拿到列表中按默认规则排序（滚动后可见被选项）。
- 多选 · 支持选择本页 / 所有数据：点「选择全部数据」后拉取所有数据 loading，期间前端已选值处于禁用态；加载完成后全部选中。

### 5.4 下拉菜单带新建　⏳

- 场景一：通过输入较少信息快速新建枚举项 —— 点击后当前浮层穿梭（大小保持一致），完成后将新增结果带回列表。
- 场景二：通过点击后跳转写填表单（侧边抽屉 / 新页面），完成后回传到下拉列表中。
