# 输入框 Input · 设计规范（给设计师）

> 这份讲"用什么、为什么、规格多少"，面向设计与评审。
> "用 antd 怎么实现"见研发文档 `input.md`，本文不涉及代码/组件名。
> 源自 Sens.Design 输入框规范 v2.1（2023.03.29）。

## 范围（第一轮）

本文只覆盖 **「文本」单行输入框**（Figma `2214:13286`）。

不在本轮：**文本域**、**数字输入框**（含步进器）——后续在同一输入框规范内追加章节。**搜索框**为独立规范，不在本文。

## 输入框是什么

以「框」为载体实现**文本录入**。常用于表单编辑态；在详情页等只读场景下，以只读或禁用样式呈现。

## 构成框架

自上而下（完整表单场景）：

1. **标题（Label）**：简短名词（如「姓名」），避免口语化和多余标点。
2. **框体（Input）**：用户录入区域。
3. **占位符（Placeholder）**：未输入时的提示。
4. **限制 / 帮助 / 反馈信息**：字数限制、帮助说明、校验结果。

非特殊说明：鼠标悬停在输入框上**不要**出 Tooltip。

## 什么时候用

- 表单、筛选条件等需要用户**输入单行文本**的场景。
- 需要明确「激活」视觉反馈的录入交互。

## 什么时候不用

| 场景 | 应使用 |
|---|---|
| 搜索（带搜索图标、即时/触发检索） | 搜索规范（`search`） |
| 多行长文本 | 文本域（后续章节） |
| 仅数字、步进调节 | 数字输入框（后续章节） |
| 前缀/后缀、币种等组合 | 组合输入框（后续或 sensd） |
| 密码掩码 | 密码输入框（后续或 sensd） |

## 尺寸

- **大 32px**（常用）/ **小 24px**（空间紧张）。
- 宽度建议 **128–600px**；尽量让占位符完整展示，不要为美观强行统一所有框宽。
- 大：字号 14px / 行高 22px；小：字号 12px / 行高 18px。
- 大圆角 4px；小圆角 3px。

## 状态

7 个语义状态 × **未输入 / 已输入** 两种内容，与 Figma 矩阵一一对齐：

默认 / 悬停 / 聚焦 / 禁用 / 禁用悬停 / 只读有背景 / 只读无背景。

- **禁用悬停 = 禁用**（悬停不再变化）。
- **聚焦**须有明确视觉感知（功能色边框 + 外环）。
- **只读有背景**：灰底、无描边。
- **只读无背景**：无底色、无描边，接近纯文本。

## 占位符

- 默认文案：**「请输入」**（或更精准的业务提示，如「请输入用户名称」）。
- 可展示最佳实践案例，但**不要**用来传达限制性信息（限制放帮助区）。
- 长度建议不超过框宽；显示不下用「…」截断，悬停 Tooltip 展示全文。
- 开始有效录入后占位符消失。
- **占位符必须是给用户看的文案，绝不能是设计稿图层名或组件名。**

## 校验与反馈

- 遵循《一致性交互规范》表单校验规范。
- 出错提示与输入框**紧密关联**；格式建议「出错原因 + 修改指导」。
- 用户**有效修改**后，错误提示消失，恢复展示限制/帮助信息。
- 校验成功**不推荐**加「对号」图标（非必要业务除外）。

### 警告展示两种方式

| 方式 | 用在哪 |
|---|---|
| **外显（推荐）** | 输入框下方展示错误文案 + 图标 |
| **框内（inside）** | 空间受限（如表达式/输入型表格）；框内右侧报错图标，悬停图标出 Tips |

框内 / 框外**边框变红规则一致**；警告色为状态色红，不随功能色换肤变。

## 清空 / 重置

- **默认不开启**。
- 建议开启：有预置默认值、或用户大概率会频繁修改。
- 出现时机：**已有输入内容** + 鼠标悬停输入框或输入框处于激活态。
- 图标位置：删除（清空）偏左；报错菱形偏右；图标间距 4px。

## 颜色语义（只记角色）

| 角色 | 行为 |
|---|---|
| 默认边框 | 中性深分割线 |
| 悬停边框 | 功能色绿（随换肤） |
| 聚焦边框 + 外环 | 功能色点击绿 + 浅色外环（随换肤） |
| 禁用 | 浅分割线边框 + 浅灰底 |
| 已输入文字 | 主要文字色 |
| 占位符 | 禁用系文字色（较浅） |
| 警告 | 状态色红（不换肤） |
