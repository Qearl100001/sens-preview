# 搜索 Search · 设计规范（给设计师）

> 这份讲"用什么、为什么、规格多少"，面向设计与评审。
> "用 antd 怎么实现"见研发文档 `search.md`，本文不涉及代码/组件名。

## 什么时候用搜索
用于**录入文本后检索、给出结果**的场景。设计原则两个字：**准确、贴心**——准确告知能搜什么、过程给反馈、结果有序、异常有提示；贴心地在全过程提供引导（历史、匹配、纠错建议等）。

## 基本构成
框体 + 搜索图标 + 占位符。三者缺一不可。

## 两种交互方式（先想清楚用哪种）
| 方式 | 用在哪 |
|---|---|
| 实时型（推荐）| 输入即动态搜索，命中内容高亮；适合即时筛选 |
| 触发型 | 输入后点按钮/回车才搜；适合开销大的检索 |

## 变体（看场景选）
- 基础搜索：单框体。
- 复合搜索：搜索框 + 范围选择（如左侧选搜索维度）。
- 简约搜索：弱化样式（仅底部分割线），用于次要位置；**与常规同为 32px 高**。

## 规格
- **高度**：**32px**（`size/component-height/m`），搜索不提供更小尺寸。
- 宽度：建议**定宽，默认 200px**，范围 108–600px；定宽时让占位符尽量完整展示。
- 有有效输入时**必须有清空/重置**，且默认展示。

## 占位符规则（容易做错）
- 文案要**准确、精简**：「搜索用户名称/用户ID」优于「搜索关键词」；最精简就是「搜索」。
- 太长时用「…」截断，悬停出 tooltip 显示全部；发生输入时占位符消失。
- **占位符是给用户看的提示文案，绝不能是设计稿里的图层名/组件名。**

## 状态
默认 / 悬停 / 聚焦 / 有输入(带清空)。**搜索无禁用态。** 若稿子是变体×状态矩阵，按矩阵逐个对，不要拍平、不要漏。

## 结果与异常
- 结果区准确展示「条目数 + 命中条目」，命中关键词高亮。
- 搜索为空 / 异常：用我们自己的空状态插图，明确告知"发生了什么、该做什么"。
