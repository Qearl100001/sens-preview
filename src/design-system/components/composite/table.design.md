# 表格 Table · 设计规范（给设计师）

> 这份讲"用什么、为什么、规格多少"，面向设计与评审。
> "用 antd 怎么实现"见研发文档 `table.md`，本文不涉及代码/组件名。

## 表格的区块构成（自上而下，缺一不可）
1. 标题区：标题 + 右侧主操作按钮（如「创建 X」）。
2. 筛选区：搜索框固定最左 + 若干筛选项；占位文案用「请+动作」。
3. 信息区：计数（共 N 条 / 已找到 N 条 / 当页选中 N 条）。
4. 表体。
5. 分页器：默认每页 20 条；左侧「本页显示第 n-m 条」。

## 列与单元格类型（设计时先把每列定好类型）
- 文本：左对齐。
- 数字（用于对比/计数）：**右对齐**。
- 超长文本：省略号 + 悬停 tooltip。
- 可排序列：表头带排序箭头；一次只排一列，点别列时上一列回默认。
- 状态列：状态点 + 文案，按语义取色。
- 操作列：见下。

## 操作列（最常被做错）
- 用**蓝色链接**形式（这是"链接"语义，状态色，不随换肤变），不是深色文字。
- 操作 ≤3 个平铺；>3 个时第 3 个起收进「更多」。
- 固定为最后一列；表格超宽时操作列冻结在右侧。
- 文案以动词为主（查看/编辑/删除）。

## 行为
- 表头吸顶：向上滚动时表头保持可见。
- 首尾冻结：水平滚动时固定首列（主属性）+ 尾列（操作）。
- 行高两套：固定高度（单行，最多 2 行后省略）/ 动态高度（内容多或含缩略图时）。

## 异常状态（必做）
- 数据为空 / 筛选为空：用我们自己的空状态插图，给明确引导。

## 计数文案
- 默认「共 n 条」；筛选后「已找到 n 条」；批量后「当页选中 n 条」。
