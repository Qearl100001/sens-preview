# 价值体验设计原则

> 来源：Sens.Design 价值体验设计原则 v2.1。本文是给 AI 使用的规则摘要，不替代原始设计文档。

## 一句话

设计最终要为客户创造价值。企业软件页面不是为了“好看”，而是为了让客户和用户更准确、更高效地完成任务。

## 价值体验设计

价值体验设计建立在充分理解客户实际业务需要之上，基于客户 / 用户双中心，以可度量价值交付为目标。

对 AI 来说，这意味着：

- 不要只生成静态页面，要说明页面解决了哪个任务。
- 不要只看单用户操作，要考虑多角色、长流程、多触点。
- 不要只追求一次生成，要让设计价值可被产品 / 研发讨论和验证。

## 四个关键原则

### 1. 准确性

信息传达的准确性是第一性原则。

AI 生成页面时必须检查：

- 信息架构是否准确。
- 用户当前位置是否清楚。
- 静态文案是否准确、无歧义、语境统一。
- 操作项是否给出自然的操作预期。
- 不可操作项是否有明确告知。
- 推荐操作 / 负向操作是否有准确的引导或预警。
- 状态反馈是否及时、可感知、可理解、可闭环。

TikTok demo 对应：

- 数据源卡片要明确显示“已接入 / 未接入 / beta”等状态。
- 连接列表要明确“启用 / 停用”等状态。
- 删除、放弃、提交后要有可理解反馈。

### 2. 高效率

工具类产品的核心指标是任务执行效率。

AI 生成页面时必须检查：

- 一个单元流程内是否只有一个核心任务目标。
- 核心任务是否从入口到完成路径清楚。
- 每个节点是否有清晰用户价值。
- 是否避免无意义创新，复用已有一致性。

TikTok demo 对应：

- 创建连接流程应从数据源卡片进入，空态引导创建，抽屉提交后回到列表。
- 抽屉中“创建”是主任务，账号映射是子任务，不能喧宾夺主。

### 3. 无障碍

这里是有限无障碍，不追求全量无障碍系统，但要保证企业办公环境下的可用性底线。

AI 生成页面时必须检查：

- 页面去色情况下，状态是否仍可分辨。
- 投影仪、共享屏、高亮环境下是否还能识别核心内容。
- 1280-1920 宽度范围内是否可用。
- 文案、按钮、状态不要只靠颜色表达。

TikTok demo 对应：

- 状态点旁必须有状态文字。
- 入口卡片 disabled / beta / 未接入不能只靠颜色。
- 表格操作和主按钮要有明确文本。

### 4. 生机

情感化设计应该恰当、克制、正向。

AI 生成页面时必须检查：

- 不要过度情感化。
- 帮助信息应触手可及、易理解。
- 空态可以有引导，但不要说教。
- 国际化适配要保障可用性底线。

TikTok demo 对应：

- 空态文案应直接告诉用户“当前没有连接，可创建连接”。
- 帮助文档、需要准备的信息应靠近创建任务。

## 页面评审口径

页面完成后，用这四个问题做一轮评审：

1. 准确性：用户是否知道自己在哪、能做什么、状态是什么？
2. 高效率：用户是否能用最短路径完成核心任务？
3. 无障碍：不依赖颜色时是否仍能理解页面？
4. 生机：帮助和反馈是否恰当、不过度？
