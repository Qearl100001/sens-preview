# 页面生成校验清单

> 用于 AI 生成或修改页面后的人工 / AI 双重检查。

## 1. 来源与规则

- [ ] 是否登记了原始资料来源。
- [ ] 是否说明页面使用了哪些 foundation。
- [ ] 是否说明页面使用了哪些组件。
- [ ] 是否说明页面涉及哪些一致性流程。
- [ ] 是否记录了无法确认的设计稿 / token / 资产缺口。

## 2. 视觉严谨性

- [ ] 颜色是否来自 token / helper，而不是硬编码。
- [ ] 间距是否来自 foundation scale 或组件规范。
- [ ] 圆角是否符合组件 / 卡片层级。
- [ ] 阴影是否符合动作反馈、浮层、卡片层级之一。
- [ ] 字号、行高、字重是否符合 typography foundation。
- [ ] 图标 / logo 是否真实、清晰、尺寸稳定。

## 3. 交互流程

- [ ] 页面核心任务是否明确。
- [ ] 创建 / 添加 / 查看 / 编辑 / 删除 / 放弃 / 移除 / 二次提示是否归类正确。
- [ ] 主操作是否唯一且突出。
- [ ] 负向操作是否有准确预警或挽留。
- [ ] 提交 / 保存后是否有成功反馈和状态闭环。
- [ ] 关闭 / 返回是否符合当前任务状态。

## 4. 企业软件可用性

- [ ] 1280-1920 宽度范围是否可用。
- [ ] 状态是否不只靠颜色表达。
- [ ] 文案是否准确、无歧义、语境统一。
- [ ] 表格 / 卡片 / 表单内容是否能扫描。
- [ ] 页面是否避免了装饰性卡片堆叠。

## 5. TikTok 数据源接入专项

- [ ] 数据源管理页入口卡片是否符合入口型卡片规则。
- [ ] TikTok 未接入 / 已接入状态是否清楚。
- [ ] 空态是否给出创建入口。
- [ ] 创建连接抽屉是否符合创建流程。
- [ ] 添加账号映射是否按添加 / 移除处理。
- [ ] 删除连接是否按删除 / 挽留处理。
- [ ] 连接列表表格是否使用 TableShell / 操作列规范。

## 6. 交付状态

- [ ] 是否跑过构建或说明未跑原因。
- [ ] 是否截图 / 录屏做过视觉检查。
- [ ] 是否更新 changelog。
- [ ] 是否标注研发接手 TODO。
