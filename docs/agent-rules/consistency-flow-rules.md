# 一致性流程规则

> 来源：Sens.Design 一致性流程规范 v2.1。本文将流程规范转成 AI 可执行规则。

## 总原则

一致性流程围绕“对象”展开。常见对象流程包括：

- 创建：从 0 到 1 创建长期对象。
- 添加：过程中的过渡性新增。
- 查看：进入稳定状态查看对象详情。
- 更多：同一容器内展开 / 收起更多信息。
- 编辑：从浏览态进入编辑态修改对象。
- 删除：把长期对象从 1 到 0 清除。
- 放弃：在创建 / 编辑中主动中止。
- 挽留：阻断删除 / 放弃，避免误操作。
- 移除：过渡性对象的轻量删除。
- 二次提示：提交 / 保存前的再次确认。

## 创建

定义：基于单一目标，从 0 到 1 创造长期性对象的任务过程。

标准流程：

```text
入口触发创建
  -> 页面 / 模态 / 抽屉输入内容
  -> 提交创建
  -> 返回入口页
  -> 成功提示
```

文案规则：

- 入口：`创建XX` 或 `创建`。
- 底部：左侧 `放弃`，非强调；右侧 `提交`，强调。
- 成功：`创建成功`。

TikTok demo：

- 创建 TikTok Ads 连接属于创建。
- 抽屉主按钮应是 `提交` 或业务确认后的等价文案。
- 提交后回到连接列表并出现新连接。

## 添加

定义：创建 / 编辑流程中的分支流程，通常是过渡性新增。

普通添加：

```text
入口触发添加 -> 原位置新增
```

首次配置：

```text
入口触发配置
  -> 页面 / 模态输入内容并保存配置
  -> 返回入口页
```

TikTok demo：

- 添加广告账号映射属于添加，不是创建长期数据源连接。
- 添加映射行应在原位置新增，不要跳出主创建任务。

## 查看

定义：针对某个对象，通过主动交互，在稳定状态下查看详情。

规则：

- 查看状态需要稳定承载容器，例如页面、抽屉、Popover。
- hover tips 不是查看。
- 查看态和编辑态要有硬区隔。
- 查看态需要明确返回 / 关闭入口。

TikTok demo：

- 查看连接详情应进入稳定承载容器。
- 表格中的“查看”不要直接进入编辑态。

## 更多

定义：同一容器内的局部展开 / 收起，不是模态。

规则：

- 展开内容与原容器同层级。
- 简单静态信息可不提供返回，但复杂信息推荐有收起路径。
- 多个更多区域不强制联动。

TikTok demo：

- 表格操作超过 3 个时可收进更多菜单。
- 更多菜单只承载操作，不负责替代查看详情。

## 编辑

定义：针对现存对象，从浏览态触发进入编辑态后修改。

规则：

- 常规编辑：入口触发编辑 -> 修改内容 -> 保存 -> 返回入口页 -> 成功提示。
- 未发生有效修改时，保存按钮建议禁用。
- 发生有效修改后，保存按钮强调。
- 所见即所得式编辑非必要不使用。

TikTok demo：

- 第一阶段可不做编辑连接；如出现编辑，必须与查看态区分。

## 删除与挽留

定义：删除长期存在对象，从 1 到 0 清除。原则上需要挽留。

流程：

```text
入口触发删除 -> Popover / Dialog 挽留
```

文案规则：

- 入口：`删除`。
- 确认：`确定删除`，警告色。
- 取消：`暂不删除`，非强调。

TikTok demo：

- 删除数据源连接属于删除，不能直接删。
- 如果连接启用中，第一版可禁用删除并给出原因。
- 如果允许删除，应先挽留确认。

## 放弃与挽留

定义：在创建 / 编辑流程中主动中止。

规则：

- 创建场景下，原则上放弃需要挽留。
- 编辑场景下，发生有效修改才需要挽留。
- 临时添加场景可不挽留。

TikTok demo：

- 创建连接抽屉中，如果用户已填写有效内容后关闭，应出现放弃挽留。
- 未填写内容可直接关闭。

## 移除

定义：过渡性对象的轻量删除，通常直接移除。

TikTok demo：

- 删除账号映射行属于移除，不是删除连接。
- hover 可变警告色，完成移除即可；复杂映射可视业务需要增加确认。

## 二次提示

定义：提交 / 保存场景下的再次确认。

普通场景：

- 是常规提交的延续。
- 可以采用引导性设计。
- 按钮：左侧放弃，右侧确定提交 / 保存。

慎重场景：

- 涉及不可逆操作、关键流程流转、重要资源占用。
- 必须采用对抗性设计，避免误操作。

TikTok demo：

- 第一阶段创建连接不需要二次提示，除非 PRD 声明会触发真实授权 / 资源占用。
- 删除连接使用删除挽留，不归类为提交二次提示。
