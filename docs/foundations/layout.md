# Layout Foundation

> 主要来源：Figma `Sens.Design 栅格布局 v0.3 220505`、`设计系统 2.1 侧导航设计`、`/Users/liyuwen/Desktop/DESIGN.md`、`src/features/tiktok-ads-connections/DataSourceManagementPage.tsx`、`src/ui/SensPageTitleBar.tsx`。
> 当前状态：第一版规则沉淀。定义页面骨架、断点和左侧区域行为；不在本文件定义栅格列数或侧导航组件 API。

## 1. 定位

Layout 负责页面区域之间的关系：顶部、左侧、标题区、主内容区如何组合，以及可展开左侧区域是否影响主内容宽度。

它不回答“卡片占几栏”或“每栏多宽”；这些属于 [Grid Foundation](./grid.md)。

## 2. 页面骨架

| 骨架 | 适用场景 | 结构 | 规则 |
|---|---|---|---|
| T 型布局 | 三级落地页、带产品壳导航的管理页 | 顶部导航 + 左侧产品壳导航 + 主内容区 | 产品壳侧导展开宽度固定为 `220px`；主内容区使用剩余可用宽度 |
| 上下布局 | 四级页面、独立管理页、下钻详情页 | 标题 / 操作区 + 下方内容区 | 标题区不承担复杂分栏，内容区再选单栏、双栏、列表或表单结构 |
| 上下布局 + 上下文侧栏 | 锚点、目录、详情辅助区 | 页面内左侧辅助区 + 主内容区 | 由页面决定是否展开；展开和收起后主内容重新计算可用宽度 |

`220px` 只表示当前产品壳侧导的展开结构尺寸，来源于侧导航 Figma 和当前产品壳实现；它不是通用页面左栏 token。

## 3. 左侧区域的行为

左侧区域都可能展开或收起，但不能因此被当作同一个组件。

| 区域类型 | 行为 | 触发方式 | 主内容是否变化 | 颜色归属 |
|---|---|---|---|---|
| Product Shell Side Navigation | Overlay | 鼠标悬停临时展开 | 不变；侧导浮在内容之上 | Navigation Theme |
| Product Shell Side Navigation | Docked | 用户锁定展开 | 变化；内容区使用剩余宽度 | Navigation Theme |
| Context Side Panel | Reflow | 锚点 / 目录 / 辅助信息展开收起 | 变化；内容区和其中栅格重新计算 | Foundation 或组件自身语义 |

约束：

- 产品壳侧导的 `Overlay` 与 `Docked` 是导航组件状态，不应由页面各自模拟。
- 锚点、目录等 `Context Side Panel` 属于页面内容，不自动复用 `theme-side-*` 颜色。
- `Grid` 只消费 Layout 算出的可用内容宽度，不决定左侧区域的宽度或交互。

## 4. 断点与适配

| 断点 | 宽度 | 页面策略 |
|---|---:|---|
| XS | 1280 | 静态布局；结构保持稳定，必要时横向滚动 |
| SM | 1366 | 桌面端常规响应式验收区间 |
| MD | 1440 | 设计画布默认宽度 |
| LG | 1600 | 大桌面内容可更舒展 |
| XL | 1920 | 保留更宽的内容展开空间 |

- 小于 `1280px` 不强行折叠或换行产品壳结构，采用横向滚动保护信息层级。
- 大于等于 `1280px` 时，页面根据实际可用内容宽度自适应。
- 左侧区域在 `Docked` 或 `Reflow` 状态下变化后，主内容和其内部栅格都应重新计算，不保留旧宽度。

## 5. 与其他 Foundation 的边界

- `Grid`：列数、占栏和局部容器排布。
- `Spacing`：区域和组件之间的 `padding`、`margin`、`gap` 数值。
- `Size`：固定尺寸档位；不负责整页区域关系。
- `Card`：卡片容器样式；不决定卡片在一行排几张。
- `TitleBar`：标题区内部结构；不决定整页骨架。

## 7. 代码落地规则

- 当前不新增独立 Layout token 生成链路。
- 产品壳侧导宽度、内容最大宽度、页面边距、双栏比例等，如需反复消费，优先判断为 `config / 常量`，不直接提升为全局 token。
- 页面选择骨架后，再由 Grid 组织内容；不要由 Card、Table、Form 反向定义整页结构。

## 8. 第一阶段验收

- 页面能明确选择 T 型、上下，或上下 + 上下文侧栏骨架。
- 需要展开 / 收起左侧区域时，能先判断 Overlay、Docked 或 Reflow。
- 低于 `1280px` 时页面不会因强行压缩而丢失结构。
