# Grid Foundation

> 主要来源：Figma `Sens.Design 栅格布局 v0.3 220505`、`Sens.Design 卡片 v2.1 20230320`、`/Users/liyuwen/Desktop/DESIGN.md`、`src/features/tiktok-ads-connections/DataSourceManagementPage.tsx`、`src/design-system/components/base/drawer.design.md`。
> 当前状态：第一版规则沉淀。定义页面主栅格、抽屉内部栅格和局部容器排布；不定义页面骨架或左侧区域交互。

## 1. 定位

Grid 负责在 Layout 已经算出的可用内容区内，组织列数、占栏关系和局部容器排布。

它回答的是：

- 页面内容的大区块如何按列组织。
- Drawer / Modal 内的表单和辅助信息如何分栏。
- 卡片、筛选区、表格区和双栏信息区如何在容器内排布。

它不回答左侧区域是否展开、是否覆盖内容，或侧导应有多宽；这些属于 [Layout Foundation](./layout.md)。

## 2. 基础规则

- 栅格最小单元使用 `4px`，与 Spacing 的基础单位保持一致。
- 页面主内容使用 `20 栏栅格`。
- Drawer / Modal 内部使用 `12 栏栅格`。
- 局部容器根据内容结构二次分栏，不机械追求等分。
- 栅格始终使用 Layout 计算后的剩余内容宽度。

## 3. 栅格层级

| 层级 | 规则 | 适用内容 | 边界 |
|---|---|---|---|
| 页面主栅格 | `20 栏` | 页面中的卡片区、筛选区、表格区、说明区 | 决定大区块宽度关系，不要求每个组件显式占栏 |
| Drawer / Modal 内部 | `12 栏` | 表单、说明、辅助信息、分组内容 | 不直接搬用整页 `20 栏` |
| 局部容器 | 按内容决定 | 卡片列表、左右双栏、映射配置区、筛选操作组合 | 不新增一套独立 foundation token |

## 4. 局部容器规则

适用：

- 卡片列表：可使用 `4` 或 `5` 列候选，但由页面容器和可用内容宽度决定。
- 左右双栏：优先根据内容密度与阅读顺序选择比例，不把比例固化到 Card Foundation。
- 筛选与操作区：可在页面主栅格内形成局部组合，不把控件间距当作栅格规则。
- Drawer 表单：先按单栏或说明 + 表单关系组织，再使用 `12 栏`细化分组。

当产品壳侧导处于 `Docked`，或目录等上下文侧栏处于 `Reflow` 状态时，局部卡片和双栏区域必须基于新的内容宽度重新排布。

## 5. 与其他 Foundation 的边界

- `Layout`：决定内容区从哪里开始、是否被左侧区域挤压，以及可用宽度。
- `Spacing`：控制列间与组件间的间距数值。
- `Card`：定义卡片的容器样式，不定义卡片一行的数量。
- `Drawer`：定义抽屉尺寸档位、圆角、投影和内容 padding；Grid 只定义内部组织方式。

## 7. 代码落地规则

- 当前不新增独立 Grid token 生成链路。
- `20 栏`、`12 栏` 和局部列数先作为规则与容器实现消费。
- 如果同一列比、最大宽度或页面 padding 在多个真实模板重复出现，再评估进入布局 config / 常量。
- 不在业务页面随意创造新的整页栅格口径。

## 8. 第一阶段验收

- 页面主内容、Drawer / Modal、局部容器能明确选择对应栅格层级。
- `20 栏` 和 `12 栏` 不会互相混用。
- 左侧区域展开或收起后，局部栅格会随可用内容宽度重新计算。
